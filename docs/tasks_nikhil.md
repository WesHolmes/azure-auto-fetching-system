
# User Disabling HTTP Triggers
1. Create a HTTP trigger for disabling a single inactive user account (single tenant, single resource)

2. Create a HTTP trigger for identifying all inactive user accounts and disabling all of them. (single tenant, multiple resource)

3. Create a HTTP trigger for identifying all inactive user accounts and disabling all of them across all tenants (multi-tenant, multiple resources)




# User Password Reset HTTP Triggers
1. Create a HTTP trigger for reseting user account (single tenant, single resource)

2. Create a HTTP trigger for reseting all user accounts. (single tenant, multiple resource)

3. Create a HTTP trigger for identifying all inactive user accounts and disabling all of them across all tenants (multi-tenant, multiple resources)
